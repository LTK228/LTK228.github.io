儲存庫(repository)無法 clone 或 push - Repository not found

Posted on 2019-10-11 Edited on 2019-10-23 In 學習筆記 Facebook: 0

今年 2019 年 Github 可以免費使用私人儲存庫囉！機會難得一定要嘗試看看，於是自己建立了私人儲存庫，卻遇到無法 clone 或 push 的情形，要怎麼解決呢？

解決方法

一、首先出問題後通常是伺服器的問題，因此要修改位置

修改遠端伺服器(remote repo)的方法有兩種：

  1. 直接修改 Git 遠端伺服器位置
  2. 先刪除，再新增
  3. 修改 _config.yml 檔案的 deploy

二、HTTPS (和 SSH 二擇一即可)

git clone https://<Github名稱>:<Github密碼>@github.com/path_to/TestPrivate.git

以上上面的圖為例，就會變成輸入成這樣

git clone https://LTK228:[Github的密碼]@github/LTK228/TestPrivate.git

三、SSH (和 HTTPS 二擇一即可)

1. 確定 Git 有輸入身份

2. 刪除 known_hosts

顯示隱藏資料 → 找到 .ssh → 刪除 known_hosts

3. 在 Git 輸入

出現以下資訊，按下 Enter

輸入兩次密碼

之後出現以下資訊代表成功

4. 在 github 上添加SSH key

在Github點選 setting → SSH and GPG keys → key
將在 /.ssh/id_rsa.pub 的檔案用文字編輯器打開，複製內容到 github，點擊 add ssh key

5. 確定

輸入命令 ssh -T git@github.com，若有出現以下內容，代表成功，
